0x00	简介
    概述：http协议是一个给予请求与响应模式的、无状态的、应用层的协议，常基于TCP的连接方式。

    1、主要特点：
        (1)无连接：指每次连接只处理一个请求，处理完之后就断开连接
        (2)无状态：指协议对于事务处理没有记忆能力，缺少状态意味着如果后续处理需要前面的信息，则必须重传，这样会导致某些连接传送的数据量增大，另一方面在服务器不需要先前信息时它的应答则较快
	(3)灵活：允许传输任意类型的数据对象，正在传输的类型由content-Type加以标记
	(4)简单快速：客户端向服务端请求服务时，只需传送方法和路径


0x01	http请求
    概述：http请求头由三部分组成，分别是请求行，消息报头，请求正文

    1、请求行：请求行以一个方法符号开头，以空格分开，后面跟着要请求的URL和协议的版本
	(1)请求方法：
	    GET：请求获取request-URL所标识的资源
	    POST：在request-URL所标识的资源后附加新的数据
	    HEAD：请求获取由request-URL所标识资源的响应消息报头
	    PUT：请求服务器存储一个资源，并用request-URL作为其表示
	    DELETE：请求服务器删除request-URL所标识的资源
	    TRACE：请求服务器回送收到的请求信息，主要用于测试和诊断
	    CONNECT：保留，将来也许会使用这个方法
	    OPTIONS：请求查询服务器的性能，或者查询与资源相关的选项和需求
    2、消息报头，见0x03
    3、请求正文（略）


0x02	http响应
    概述：服务器接收一个请求后，就会返回一个响应消息，响应消息也由三个部分组成，分别是状态行、消息报头、响应正文

    1、状态行：一个由三位数字组成的代码，由第一个数字定义响应的类别
	(1)1xx：提示消息，表示请求接收，等待处理
	(2)2xx：连接成功，表示请求已被成功接收、理解、接手
	(3)3xx：重定向，表示要完成请求必须进一步的操作
	(4)4xx：客户端错误
	(5)5xx：服务端错误
    常见状态码：
	200	ok			//客户端请求成功
	400	Bad Request		//客户端请求有语法错误，不能被服务器接收
	401	Unauthorized		//非法请求，无法授权
	403	Frobidden		//服务器收到请求，但是拒绝提供服务
	404	Not Found		//请求资源不存在
	500	Interal Server Error	//服务器发生错误
	503	Server Unavailable	//服务器当前不能处理客户端请求
    2、响应报头：见0x03
    3、响应正文：指服务器返回的资源的内容，一般为html


0x03	消息报头
    概述：http消息由客户端到服务器的请求和服务器到客户端的响应组成，请求消息和响应消息都是由开始行（请求行和状态行），消息报头，空行，消息正文组成

    1、普通报头：在普通报头中，有少数报头域用于所有的请求和响应消息，但并不用于被传输的实体，之用语传输的消息
	(1)Cache-Control：指定缓存指令
	    请求时的指令：no-cache、no-store、max-age、max-stale、min-fresh、only-ifcached
	    响应时的指令：public、private、no-cache、no-store、no-transform、must-revalidate、proxy-revalidate、max-age、s-maxage
	(2)Date：普通报头域中表示消息产生的日期和时间
	(3)Connection：普通报头域允许发送指定连接的选项，比如常见的“close”，指定完成响应后关闭连接

    2、请求报头：客户端向服务端传递请求的附加信息以及客户端自身的信息
	(1)Accept：请求报头域中，用于指定客户端接受哪些类型的信息
	(2)Accept-Charset：请求报头域中，用于指定客户端接受的字符集
	(3)Accept-Encoding：请求报头域中，用于指定可以接受的内容编码
	(4)Accept-Language：请求报头域中，用于指定自然语言
	(5)Authorization：请求报头域中，用于证明客户端有权查看某个资源
	(6)Host：请求报头域中，用于指定被请求资源的主机和端口号
	(7)User-Agent：告诉服务端发出请求的客户端的操作系统，浏览器和其他属性
    
    3、响应报头：服务端向客户端传递的不能放在状态行中的附加响应信息，以及关于服务器的信息和对request-URL所标识的资源进行下一步访问的信息
	(1)Location：响应报头域中，用于指定重定向的位置
	(2)Server：响应报头域中，与User-Agent对应，是服务器的软件信息
	(3)WWW-Authenticate：响应报头域中，必须被包含在401响应消息中，当客户端收到401响应时，就会发送Authorizetion给服务端验证，通过则授权
	(4)Pragma：响应报头域中，用于向浏览器传送缓存指令

    4、实体报头：请求和响应消息都可以传送一个实体，一个实体由实体报头域和实体正文组成，实体报头定义了关于实体正文和请求所标识的资源的元信息
	(1)Content-Encoding：指定实体正文的附加内容的编码
	(2)Content-Language：指定实体报头描述所用资源的自然语言
	(3)Content-Length：指定实体正文的长度，以字节按十进制数字标识
	(4)Content-Type：指定发送给接收者的实体正文的类型
	(5)Last-Modified：资源的最后修改日期和时间
	(6)Expires：指定给出响应过期的日期和时间（未过期前，刷新页面时是直接从缓存中加载的，对于非法的日期格式，全部视为已过期）


That's all.
